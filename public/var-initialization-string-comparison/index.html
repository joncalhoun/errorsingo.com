undefined: firstVar

go build → "undefined: firstVar"

What causes the error?

This error is easily caused when there is mix up between variable short declaration and assignment operator.

func main() {
  // this line cause a compilation error
	firstVar=“a”
}

Run it on the Go Playground → https://play.golang.org/p/1bfb7NxBM5Q

How can I fix it?

The simple solution here is to use := operator.

Now that compilation issue is fixed, there is a logical issue in this code. With == operator, string comparison is case aware. This program wouldn't print: "firstVar and secondVar are equal."

Fix for logical issue: Use strings.Compare if you expect case-insensitive comparison.
